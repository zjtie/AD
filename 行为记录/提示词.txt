请为我生成一个现代化的单网页应用（HTML + CSS + JavaScript），用于自我管理与行为记录，适配手机与 PC 浏览器。要求如下：

📌 应用定位：
- 用于个人行为记录、自我反思与目标管理
- 支持记录添加、查看、编辑、删除
- 界面美观、排版合理，适合长期使用
- 使用先进前沿的前端技术栈
- 在 Android 手机上通过 Termux + Python 本地服务器运行（访问地址：http://localhost:7668）
- 支持云端同步：通过 WebDAV 协议直接与 AList 服务通信

🧠 功能模块：

### 1. 📋 行为记录表单（空间优化 + 结构化）
- 字段包括：
  - 时间区间（开始时间 / 结束时间 / 耗时）放在同一行展示，节省空间
    - 开始时间 / 结束时间为24小时制
    - 耗时为结束时间减开始时间，支持跨天
    - 点击时间文本触发时间填入（取消传统按钮）
  - 行为类型（下拉选择，如：学习、工作、娱乐、反思）
  - 标签（支持多标签输入与自动补全）
  - 事件弹窗：点击触发模态框，录入结构化行为信息，包括：
    - 标记（主分类）
    - 目标（行为目的）
    - 量化（可衡量指标）
    - 原因（动机或背景）
  - 日期（自动填入）
  - 完成度（滑块默认50%）
  - 回顾总结（多行文本）
    - 用于记录关键过程、反思点、情绪波动等
  - 图片上传支持点击选择与拖拽，生成本地预览，展示于记录卡片中
    - 触发式展开，默认隐藏上传区域
    - 支持点击选择与拖拽上传
    - 本地预览图展示于记录卡片中
    - 支持删除与替换操作
    
-📋 表单布局（按行分组）
  - 第1组（浅色背景 A）：
    - 行1：开始时间 / 结束时间 / 耗时（同行展示）
    - 行2：行为类型 / 标签（同行展示）
    - 行3：日期 / 事件弹窗（同行展示）
  - 第2组（浅色背景 B）：  
    - 行4：完成度（滑块）
    - 行5：回顾总结（多行文本）
    - 行6：图片上传（默认折叠，点击展开）
  📋 表单结构（移动端优先）：
    - 分为两组字段，分别使用浅色背景区分（bg-zinc-50 / bg-white）
    - 每组包含三行字段，每行使用 grid 布局，字段同行展示
    - 在 sm 以下断点自动切换为单列布局，保持紧凑
  -🧩 交互行为：
    - 点击时间文本自动填入当前时间
    - 图片上传区域默认折叠，点击按钮展开
    - 弹窗支持 Esc 关闭，Ctrl+Enter 提交


### 2. 📖 记录查看与编辑
- 卡片式布局，每条记录展示日期、时间区间、行为类型、标签、完成度、耗时
  -耗时展示分钟(4分)，超过小时展示十进制小时(1.5时)
- 支持展开查看详情，包括事件信息、回顾总结、图片预览
  -点击卡片 展开卡片/收缩卡片
  -点击卡片图片，支持图片查看
  -标记，目标，量化，原因，这8个汉字需要独立样式，目的方便阅读
- 每条记录右上角提供编辑与删除按钮
  - 默认隐藏，仅在卡片获得焦点或 hover 时显示，避免挤占空间
- 顶部提供筛选工具（日期范围、行为类型、标签关键词）
  - 日历视图（按天查看行为记录）,- 面板为浮动面板，小视图 + 自适应
  - 节省空间，移动端默认隐藏
  - 节省空间，最小视觉化设计
-  顶部提供数据操作工具(导入、导出、备份、清空)
  -	支持导出为 JSON 或 CSV，CSV支持中文
  - 支持导入 JSON 文件并解析 Unicode 编码（如 \uXXXX）
  - 面板为浮动面板，小视图 + 自适应

### 3. 🎨 UI/UX 要求
- 使用现代前端框架（Vue3、React 或 Svelte）
- 使用美观的组件库（Tailwind CSS、Material UI 或 Ant Design）
- 响应式设计，适配手机与桌面浏览器
- 支持暗黑模式切换
- 支持快捷键操作（如 Ctrl+Enter 提交、Esc 关闭弹窗）
- 时间字段为点击式交互，避免按钮堆叠
- 弹窗录入行为事件，结构清晰、字段可配置
- 图片支持预览、删除、重新上传
- PC端 行为记录表单占左边，记录查看与编辑占右边

### 4. 📦 本地数据存储
- 使用 IndexedDB（首选）或 localStorage 保存记录
- 页面加载时自动读取并渲染已有记录
- 提供“清空数据”、“导入数据”、“导出数据”功能
- 所有数据默认仅保存在本地，除非用户主动开启云同步
- 提供本地备份
- 提供备份恢复

### 5. ☁️ 云端存储与同步（WebDAV 对接 AList）
- 同步目标：AList 的 WebDAV 端点（如 https://your-alist.example/webdav）
- 设置面板：
  - WebDAV 地址
  - 用户名 / 密码 或 Token
  - 根目录（默认：/behavior-journal/）
  - 设备标识（自动生成 UUID）
  - 同步模式（手动 / 保存即同步 / 定时同步）
- 云端文件结构：
  - 记录清单：/behavior-journal/{device-id}/records_YYYY-MM.json
  - 图片：/behavior-journal/{device-id}/images/{ISODate}/{uuid}.ext
  - manifest.json：记录版本、最后同步时间、哈希
- 同步策略：
  - 离线优先：本地为主，云端为备份
  - 写入：本地立即保存，根据同步模式推送至云端
  - 冲突处理：以更新时间戳为准，保留冲突副本并提示合并
  - 批量上传、失败重试、进度反馈
- 安全建议：
  - 推荐使用 AList 专用子账号或应用密码
  - 凭据默认保存在 sessionStorage，用户可选择“记住我”后写入 localStorage

### 6. 📊 扩展功能
- 图表统计模块（如每周耗时分布、标签热度）
- 日历视图（按天查看行为记录）
- Markdown 支持（用于回顾总结）
- 定时提醒功能（如每日记录弹窗）

###  暗黑模式支持
- 一键切换按钮，自动适配 Tailwind 的 dark class
- 所有组件在暗色背景下保持清晰可读，边界分明，符合长时间阅读原则

###  视觉风格建议
- 色彩风格偏好：暗黑丝绒、留白、磨砂材质感与光影氛围
- 使用 Tailwind CSS 或类似工具，保持一致性与响应式
- 组件边界清晰，使用圆角（rounded-xl）、阴影（shadow-sm）、浅色边框（border-zinc-200 / dark:border-zinc-800）
- 字体建议使用现代无衬线字体，字号层次分明
- 图标建议使用 Heroicons 或 Material Icons

📡 部署方式：
- 在 Termux 中运行：`python3 -m http.server 7668`
- 在手机浏览器访问：`http://localhost:7668`

📁 文件结构建议：
- index.html：主页面结构
- style.css / Tailwind：样式定义
- app.js / main.js：行为逻辑与数据处理
- assets/：图标、字体、上传图片等资源
- lib/webdav.min.js：WebDAV 客户端库（如使用）

🔧 质量与维护建议：
- 模块化代码结构，结构化注释
- 错误处理与用户提示（网络错误、认证失败、同步冲突等）
- 日志面板：显示最近同步状态、失败原因、上次时间
- 工具函数：时间计算、跨日修正、UUID、debounce、哈希等
